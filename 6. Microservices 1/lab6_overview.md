# Лабораторная работа N 6
### Задание к выполнению лабораторной работы.

В этой лабораторной работе предлагается реализовать заполнение полей `name_en` и `country_en` сообщений `Place`.  
В протоколе эти поля уже есть, но Geo-сервис сейчас оставляет их пустыми.

### Требования к результату

Необходимо сделать так, чтобы поля `name` и `country` по-прежнему содержали названия на "родном" языке,
а в дополнение к ним поля `name_en` и `country_en` содержали названия на английском языке.

### Пример

Пример диагностического вывода поиска города с названиями на обоих языках:

```
2025-08-07 18:43:53.419462 Выполняем запрос:
name: "Zelenograd"
include_details: false

2025-08-07 18:43:54.828576 Получен ответ:
cities {
  name: "Зеленоград"
  name_en: "Zelenograd"
  country: "Россия"
  country_en: "Russia"
  center {
    latitude: 55.991971
    longitude: 37.214241
  }
}

2025-08-07 18:43:54.828638 Запрос выполнен.
```

Пример диагностического вывода поиска регионов с названиями на обоих языках:

```
2025-08-07 14:05:04.971484 Выполняем запрос:
position {
  latitude: 41.116525
  longitude: 1.257839
}
distance_km: 50
prefs {
  mask: 1
}

2025-08-07 14:05:06.628634 Получен ответ:
regions {
  name: "Catalunya"
  name_en: "Catalonia"
  country: "España"
  country_en: "Spain"
  center {
    latitude: 41.8523094
    longitude: 1.5745043
  }
}

2025-08-07 14:05:06.628724 Запрос выполнен.
```

### Советы по реализации

Информация о названиях локации и страны на английском языке доступна через тот же API, через который доступна информация и на родном языке - Nominatim API.  
Вам не нужно использовать Overpass API в этой лабораторной работе.

HTTP запросы, которые выполняет Geo-сервис не содержат ни одного из этих способов указать язык:

- HTTP-заголовок Accept-Language
- Параметр запроса accept-language в URL

Это дает возможность получать результаты в "родном" для объекта языке, и отдавать их пользователю в полях `name` и `country` сообщений `Place`.
